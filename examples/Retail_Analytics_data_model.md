# Qlik data model — "Retail Analytics" (as provided from Qlik)

Star schema, 2 facts + 6 conformed dimensions. Physical source:
DB_CASESTUDY.QLIKMIG_DEMO (Snowflake).

## Facts
- FACTSALES (FACT_SALES): SALES_KEY, DATE_KEY, CUSTOMER_KEY, PRODUCT_KEY, STORE_KEY,
  CHANNEL_KEY, PROMO_KEY, ORDER_NUMBER, QUANTITY, UNIT_PRICE, DISCOUNT_AMOUNT,
  SALES_AMOUNT, COST_AMOUNT, PROFIT_AMOUNT
- FACTRETURNS (FACT_RETURNS): RETURN_KEY, DATE_KEY, CUSTOMER_KEY, PRODUCT_KEY,
  STORE_KEY, RETURN_NUMBER, QUANTITY_RETURNED, REFUND_AMOUNT, REASON

## Dimensions (conformed)
- DIM_DATE   (DATE_KEY)      : FULL_DATE, DAY, MONTH, MONTH_NAME, QUARTER, YEAR, WEEK_OF_YEAR, IS_WEEKEND
- DIM_CUSTOMER (CUSTOMER_KEY): CUSTOMER_ID, GENDER, SEGMENT, REGION, COUNTRY
- DIM_PRODUCT (PRODUCT_KEY)  : PRODUCT_ID, PRODUCT_NAME, CATEGORY, SUBCATEGORY, BRAND, LIST_PRICE, UNIT_COST
- DIM_STORE  (STORE_KEY)     : STORE_ID, STORE_NAME, STORE_TYPE, REGION, COUNTRY
- DIM_CHANNEL (CHANNEL_KEY)  : CHANNEL_NAME
- DIM_PROMOTION (PROMO_KEY)  : PROMO_NAME, PROMO_TYPE, DISCOUNT_PCT

## Associations (joins)
- FACT_SALES   -> DIM_DATE, DIM_CUSTOMER, DIM_PRODUCT, DIM_STORE, DIM_CHANNEL, DIM_PROMOTION
- FACT_RETURNS -> DIM_DATE, DIM_CUSTOMER, DIM_PRODUCT, DIM_STORE

## Key measures (inline in charts)
- Total Sales = Sum(SALES_AMOUNT); Total Profit = Sum(PROFIT_AMOUNT)
- Units Sold = Sum(QUANTITY); Total Refunds = Sum(REFUND_AMOUNT)
- Profit Margin % = Sum(PROFIT_AMOUNT)/Sum(SALES_AMOUNT)*100
